# 开发阶段操作指南

适用目录：项目根目录 `e:\blog\weblog`

这份文档只用于日常开发阶段，目标是：
- 每天快速启动前端
- 每天快速启动后端
- 每天快速启动 MinIO
- 出问题时快速排查

---

## 一、每天开发前先确认 3 个东西

1. MySQL 已启动
2. MinIO 已启动
3. 前后端端口没被占用

默认端口：
- 前端：`5173`
- 后端：`8080`
- MinIO API：`9000`
- MinIO Console：`9001`

## 三、MinIO 启动

你当前开发配置里 MinIO 是：
- endpoint: `http://127.0.0.1:9000`
- accessKey: `minioadmin`
- secretKey: `minioadmin`
- bucket: `weblog`

### 方式 1：如果你本机已经装了 Docker，直接启动 MinIO

```powershell
docker run -d --name minio-dev -p 9000:9000 -p 9001:9001 -e MINIO_ROOT_USER=minioadmin -e MINIO_ROOT_PASSWORD=minioadmin -v E:\blog\minio-data:/data quay.io/minio/minio server /data --console-address ":9001"
```

### 方式 2：如果之前已经创建过容器，直接启动

```powershell
docker start minio-dev
```

### 检查 MinIO 是否成功启动

浏览器打开：

```text
http://127.0.0.1:9001
```

登录账号：

```text
minioadmin
```

登录密码：

```text
minioadmin
```

如果桶 `weblog` 不存在，就手动新建一个：

```text
weblog
```

## 七、如果启动失败，优先排查这几项

## 1）端口被占用

### 查 8080 端口

```powershell
netstat -ano | findstr :8080
```

### 查 5173 端口

```powershell
netstat -ano | findstr :5173
```

### 查 9000 端口

```powershell
netstat -ano | findstr :9000
```

### 强制结束某个进程

把下面的 `PID` 换成你查到的数字：

```powershell
taskkill /PID PID /F
```

---

## 2）MinIO 没启动或桶不存在

### 查看 MinIO 容器状态

```powershell
docker ps -a | findstr minio-dev
```

### 启动 MinIO

```powershell
docker start minio-dev
```

### 如果容器坏了，先删掉再重建

```powershell
docker rm -f minio-dev
docker run -d --name minio-dev -p 9000:9000 -p 9001:9001 -e MINIO_ROOT_USER=minioadmin -e MINIO_ROOT_PASSWORD=minioadmin -v E:\blog\minio-data:/data quay.io/minio/minio server /data --console-address ":9001"
```

---

## 3）后端启动失败

常见原因：
- MySQL 没开
- 数据库 `weblog` 不存在
- 数据库账号密码不对
- MinIO 没开
- 端口 8080 被占用

### 检查数据库配置（当前 dev 环境）

```text
host: localhost
port: 3306
database: weblog
username: root
password: root
```

### 如果数据库没初始化，先导入 SQL

SQL 文件位置：

```text
E:\blog\weblog\weblog-springboot\weblog-web\src\main\resources\db\weblog.sql
```

---

## 4）前端启动失败

### 删除依赖后重装

```powershell
cd E:\blog\weblog\weblog-vue3
rmdir /s /q node_modules
del package-lock.json
npm install
npm run dev
```

如果你是在 PowerShell 里执行 `rmdir` 不方便，也可以直接删目录后再执行：

```powershell
npm install
npm run dev
```
