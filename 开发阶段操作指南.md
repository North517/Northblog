# 二、每天开发前先确认的 4 件事

每天正式开发前，先确认下面 4 件事：

1. MySQL 已启动
2. MinIO 已启动
3. 后端端口 `8080` 没被占用
4. 前端端口 `5173` 没被占用

如果这 4 个都正常，后面的启动一般就会顺很多。

---

# 三、当前项目推荐启动方式

这个项目当前已经验证过的稳定启动方式如下：

- MinIO：本地直接运行 `minio.exe`
- 后端：先在根工程执行 `mvn install -DskipTests -pl weblog-web -am`，再运行 jar
- 前端：进入 `weblog-vue3` 执行 `npm run dev`

注意：

## 不要再默认使用旧写法直接在 `weblog-web` 目录里执行 `mvn spring-boot:run`

这个项目是多模块 Maven 工程，之前已经验证过，这样做容易因为父工程版本 `${revision}`、主启动类识别等问题导致失败。

# 四、MinIO 启动方式

当前项目开发环境中的 MinIO 配置为：

- endpoint：`http://127.0.0.1:9000`
- accessKey：`minioadmin`
- secretKey：`minioadmin`
- bucket：`weblog`

## 推荐方式：本地启动 MinIO（不用 Docker）

前提：你本机已经有下面这个文件：

```powershell
E:\minio\minio.exe
```

### 启动命令

```powershell
cd E:\minio
.\minio.exe server E:\minio-data --console-address ":9001"
```

### 启动后访问

#### MinIO API

```text
http://localhost:9000
```

#### MinIO 控制台

```text
http://localhost:9001
```

#### 账号密码

```text
账号：minioadmin
密码：minioadmin
```

### 桶（bucket）要求

如果 `weblog` 这个桶不存在，需要登录 MinIO 控制台后手动创建：

```text
weblog
```

## 不推荐作为当前默认方式：Docker 启动 MinIO

之前已经遇到过：

- Docker Desktop 没开
- `docker_engine` 管道错误
- 权限不足

所以当前项目默认开发阶段，优先使用本地 `minio.exe` 启动。

如果以后你重新启用 Docker，再考虑恢复 Docker 方式。

# 九、常见问题与解决方法

## 1）后端 8080 端口被占用

### 检查 8080

```powershell
netstat -ano | findstr :8080
```

### 杀掉占用进程

```powershell
taskkill /PID 进程号 /F
```

然后再重新启动后端。

---

## 2）前端 5173 端口被占用

### 检查 5173

```powershell
netstat -ano | findstr :5173
```

### 杀掉占用进程

```powershell
taskkill /PID 进程号 /F
```

然后重新执行：

```powershell
cd E:\blog\weblog\weblog-vue3
npm run dev
```

---

## 3）MinIO 9000 / 9001 端口被占用

### 检查 9000

```powershell
netstat -ano | findstr :9000
```

### 检查 9001

```powershell
netstat -ano | findstr :9001
```

### 杀掉占用进程

```powershell
taskkill /PID 进程号 /F
```

然后重新启动 MinIO。

## 4）后端启动时报数据库错误

优先排查：

- MySQL 是否启动
- `weblog` 数据库是否存在
- 账号密码是否还是 `root/root`
- 数据库表是否初始化完成

## SQL 初始化文件位置

```text
E:\blog\weblog\weblog-springboot\weblog-web\src\main\resources\db\weblog.sql
```

如果是全新数据库，先导入这份 SQL。

---

## 5）后端启动时报 MinIO 错误

优先排查：

- MinIO 是否已启动
- `http://localhost:9001` 是否能打开
- `weblog` 桶是否存在
- 账号密码是否仍然是 `minioadmin / minioadmin`

---

## 6）后端启动时报 `Port 8080 is already in use`

这说明后端实际上已经开始启动了，只是最后端口被占了。

处理方式仍然是：

```powershell
netstat -ano | findstr :8080
taskkill /PID 进程号 /F
```

然后重新启动后端。

---

## 7）PowerShell 里 `rmdir /s /q` 报错

这是因为：

- `rmdir /s /q` 是 `cmd` 写法
- PowerShell 下不完全兼容

### PowerShell 正确写法

```powershell
Remove-Item 目录名 -Recurse -Force
```

比如删除 `.git`：

```powershell
Remove-Item .git -Recurse -Force
```

如果你非要用 `cmd` 方式，也可以：

```powershell
cmd /c "rmdir /s /q .git"
```

## 8）前端依赖坏了，前端启动失败

### 进入前端目录

```powershell
cd E:\blog\weblog\weblog-vue3
```

### PowerShell 下推荐删除方式

```powershell
Remove-Item node_modules -Recurse -Force
Remove-Item package-lock.json -Force
npm install
npm run dev
```

---

## 9）Git 推送失败

如果 `git push` 失败，优先检查：

- 网络是否正常
- 当前远程仓库地址是否正确
- SSH 是否已经配置成功
- GitHub 是否能正常连接

### 查看远程地址

```powershell
git remote -v
```

如果以后还是用 SSH，远程地址通常类似：

```text
git@github.com:你的用户名/仓库名.git
```
